HTB Command Injection writeup

August 4, 2025 • 6 min read

Concepts covered

Command injection Filter bypass Blacklist bypass Obfuscation Space bypass

Step 1: Authenticate as Guest

We need a valid session to test file operations. The credentials guest:guest are provided.

CI3.png confirms successful login: we see our home directory file listing and the actions panel.

Step 2: Examine Allowed Actions

The “Actions allowed” panel reveals we can copy files and invoke a function on each file.

Step 3: Write to tmp Directory

We test writing into /var/www/html/files/tmp, which succeeds without restrictions.

Step 4: Path Restriction Enforcement

Writing outside the tmp folder (e.g. into /tmp) is blocked as malicious.

Step 5: Identify Injection Point

Intercepting the move request shows the target path is sent via URL parameters after index.php. We can append shell operators here.

Step 6: Operator Reference

Various shell operators behave differently when URL-encoded. We use the ampersand (&) to run a second command.

Operator Character URL-encoded Behavior
Semicolon;%3bboth commands
New line\n%0aboth commands
Background&%26both; second output first
Pipe|%7conly second output
AND&&%26%26second if first succeeds
OR||%7c%7csecond if first fails
Backticks``%60%60both (Linux)
$()$()%24%28%29both (Linux)

Step 7: Blacklist Bypass

The simple whoami is blocked. We obfuscate it with quotes and whitespace expansion: 

?to=tmp$IFS%26w'h'oa'm'i&from=78113764.txt&finish=&move=

Step 8: Directory Enumeration

We list home directory entries, including hidden files: 

?to=tmp$IFS%26l's'$IFS'-a'${HOME:0:1}&from=787113764.txt&finish=&move=

Step 9: Flag Extraction

Finally, swap to ${PATH:0:1} to target the root directory and read the flag: 

?to=tmp$IFS%26l's'$IFS'-a'${PATH:0:1}&from=787113764.txt&finish=&move=
