HTB SQL Injection Fundamentals writeup

August 2, 2025 • 5 min read

Step 1: Initial Login Panel

We’re presented with a login panel as our initial attack surface. This is often a prime candidate for SQL Injection testing because it sends user-controlled data directly to the backend for authentication.

Step 2: Capturing the Request

Using Burp Suite, we intercept the login request to analyze the parameters. This helps us determine whether the username or password fields are injectable.

Step 3: Testing for SQL Injection

We insert a single quote (`'`) followed by a comment operator to intentionally break the query and observe differences in the response. A blank or error-filled page can be an indicator of vulnerability.

Step 4: Successful Authentication Bypass

Our crafted payload successfully bypasses authentication, granting us access to the application. This confirms the presence of SQL Injection.

Step 5: Identifying Column Count

Determining the number of columns in the query is critical for UNION-based SQL Injection. We incrementally adjust the number of NULL values until the query executes without error.

Step 6: Extracting Data

With the column count known, we can use UNION SELECT to extract sensitive data from the database. In this example, we retrieve usernames and passwords from the `users` table.

Step 7: Conclusion

This challenge demonstrates the importance of proper input validation and parameterized queries. Even basic login forms can be exploited if they pass unsanitized input directly into SQL statements.
